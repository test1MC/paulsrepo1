Creating Projects

You probably already have a Flare project that you want to convert to Side Navigation output. For the purposes of this tutorial, we are going to create two Flare projects to illustrate how you might convert your project to one that produces Side Navigation output. Here are the projects we will create:

  • Legacy Project We will create a Flare project that uses the Top Navigation format, pretending it is your existing project that you want to convert. In an actual conversion process, you will not need to perform these steps, because you will just use your existing project.
  • Temp Project We will also create a simple Flare project based on the Side Navigation template. This will be a temporary project. Its only purpose is that we will use it to import into the legacy project the files that we need. In an actual conversion process, you will perform these steps.
Create the Legacy Project
  1. Select File > New Project. The Start New Project Wizard opens.
  2. In the Project name field, name your project Legacy.

  3. (Optional) In the Project folder field, you can enter a location where you want to save your project. By default, projects are saved in your Documents > My Projects folder.

  4. Click Next.

  5. Under Factory Templates, expand the Online folder and select the Top Navigation template.

  6. Click Next.

  7. Click Finish. The project is created and loaded into Flare.

 Create the Temp Project
  1. Open a new session of Flare. This way you can have both projects open at the same time.
  2. Select File > New Project. The Start New Project Wizard opens.
  3. In the Project name field, name your project Temp.
  4. In the Project folder field, use the same folder where your legacy project is located.
  5. Click Next.

  6. Because Flare's Side Navigation project template contains some elements that we wanted to use (e.g., the look and feel of the Home page), we will borrow those pieces and make changes to them in our legacy project. Under Factory Templates, expand the Online folder and select the Side Navigation template.

  7. Click Next.

  8. Click Finish. The project is created and loaded into Flare.

 Edit the H1 Style in the Temp Project

We could stop right here and move on to the next lesson, but we’re going to do one more thing. You will notice that these two projects look somewhat similar, but not exact (including the Home page). Let’s make a change to the color of the first-level headings in the Temp project, so that it’s a little easier to see the difference between the projects

  1. Open the Temp project.

  2. In the Content Explorer, double-click Home.htm. The topic opens to the right in the XML Editor.

  3. Scroll down until you see the heading “Latest Updates,” followed by another with the text “Frequently Asked Questions.” Each of these is using an h1 style, meaning they are first-level headings.

  4. Make sure your structure bars are turned on. These are the bars that you see to the left of the topic. If they are not on, click in the toolbar at the very bottom of the XML Editor. See About Structure Bars.

    You should notice an h1 bar to the left of each of these headings.

  5. Right-click either of these structure bars, and from the context menu select Style Class > Edit Style Class.

  6. The Stylesheet Editor opens, with the h1 style selected on the left. Make sure the other options shown in the image below are selected. If so, you should see style property called “color” that is set to a value of “var(--Dark-Gray).”

    The value “var(--Dark-Gray)” looks somewhat cryptic. This is called a CSS variable. It’s a way to set a value (such as yellow or red) in one place in the stylesheet instead of setting it in multiple places. We’re going to replace this CSS variable with another one that is a shade of blue.

  7. Make sure the color row is selected. Then, from the CSS Variable drop-down in the local toolbar, click Insert CSS Variable.

    The Insert CSS Variable dialog opens.

  8. From the drop-down, you can select any of the other CSS variables in the stylesheet. The first one in the list (--Brand) is shown by default in the field.

    This is the one we want. So just click OK.

    Notice that the new CSS variable is now shown in the color row.
