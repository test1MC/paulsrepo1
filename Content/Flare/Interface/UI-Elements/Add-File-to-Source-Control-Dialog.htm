Add File to Source Control Dialog

If you apply a condition tag to a folder in Flare, a PROPS file is automatically created behind the scenes. This file enables Flare to remember the condition tag that has been applied. If the Flare project is bound to a source control program when you apply the condition tag to the folder, the Add File to Source Control dialog displays. This dialog lets you determine whether to add the new .props file to source control (which is what you will probably do most of the time). See About Source Control and Applying Condition Tags to Content.

  1. Bind the project to source control. See Binding a Project to Source Control.
  3. Locate and click on the folder to which you want to apply a condition tag.
  4. In the local toolbar, click . The Properties dialog opens.
  5. Click the Conditional Text tab. The first condition tag set is selected and the associated condition tags are shown on the right.
  6. If you want to see condition tags for a different condition tag set, select it.
  7. For each condition tag that you want to apply to the folder, click the check box next to the tag. A check mark appears in the box.

Button/Section

Description

The following file was created…

Displays the path of the behind-the-scenes .props file that has been created.

Add file to Source Control

Adds the new .props file to source control. This is the recommended option.

Do nothing

Closes the dialog and does not add the .props file to source control.

Make this the default from now on

If you select this option, will not prompt you with this dialog in the future when you apply condition tags to folders. If you later decide that you want to be prompted again, you can open the Options dialog , choose the Source Control tab, and select Show prompt to add auto-created files to source control.
