Elasticsearch is enabled by two web services, one used by Flare and the other used by Elasticsearch. The Elasticsearch web service runs a portal that lets you manage the targets, topics, and indexes employed by the Elasticsearch engine in your server-based HTML5 target.

General Steps for Using Elasticsearch

These are the steps needed to set up Elasticsearch.

  1. Set Up Server Configure a remote server to store your search indexing data. This is usually done by a network administrator. See Installing Elasticsearch on a Remote Server.
  2. Set Search Engine Select Elasticsearch as the search engine in the Target Editor. See Enabling Elasticsearch Through the Target Editor.
  3. Create Destination Files Add and configure primary and Elasticsearch destination files so you can publish your output and search results. See Creating Publishing Destinations for Elasticsearch.
  4. Connect Destinations with Target Associate your publishing destinations with the HTML5 target. See Associating Elasticsearch Destinations with a Target.

  5. Generate and Test Output Build the HTML5 target. When the build is near completion, you will notice a new icon display in the system tray. This is for the MadCap Process Manager, which will handle the web services required for Elasticsearch if you are running locally on your machine. See MadCap Process Manager.

    Once the build is complete, start testing your output with a few searches to ensure Elasticsearch is working in accordance with your configuration.

  6. Manage Search Indexes As your Elasticsearch output is searched, you may want to control the size of the indexes for your target. See Content Services Portal.

You can also modify the appearance of your auto-complete suggestions in the search results. See Headers in HTML5 Skins and HTML5 Skin Styles.See the Flare Skins Guide or the Flare online Help for more information.

Synonyms are not supported in Elasticsearch.

If you are viewing your generated Elasticsearch output on Microsoft Edge, you will not be able to search. In order to use Edge for your Elasticsearch output, you will need to run the following command prompt one time as an administrator:

  • CheckNetIsolation LoopbackExempt -d -n=Microsoft.MicrosoftEdge_8wekyb3d8bbwe

This will disable loopback in your Edge browser and allow you to use your Elasticsearch target in Edge. For more information on this, visit Enabling Elasticsearch Targets In Microsoft Edge.
