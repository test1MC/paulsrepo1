Using Private URLs

How to Use a Private URL
  4. In the grid, click the build row (not the check box) that has the private URL you want to retrieve. The row expands, showing additional details.

  5. Copy the URL from the Private URL field.

  6. Provide this URL to anyone on your Central license.
 Tip

If you create a target vanity (see Setting Target and Build Vanities), you can use that text in place of the auto-generated number that is normally part of a private URL. The benefit of doing this is that anyone referencing your private URL will always see the most recent build for that target, instead of having to use a different private URL each time you generate a new build.

For example, you might have a private URL for a build, and the end of that URL looks something like this:

…/fictionsoft/cfb33266bffa45048d636878f00b1317/Default.htm

If you have a target vanity called “module1,” you could replace the auto-generated number. That way, you could give others a URL, the end of which looks more like this:

…/fictionsoft/module1/Default.htm

If anyone uses this second URL that contains the target vanity, the most recent build in the target will always open in the browser.

 Note

If a particular build is set as “Live,” users do not need to be logged in to Central in order to view the build’s private URL.
