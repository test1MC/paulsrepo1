 How to Schedule a Target to be Generated

  4. In the toolbar of the Targets pane, click , and from the menu, select Schedule Build.

    The Schedule Build dialog opens.

  5. Complete the fields in the dialog:

    Field

    Description

    Time

    Click in this field to open a clock popup.

    On the clock, click any of the numbers to set the hour.

    After this, the minutes area field is automatically selected. So click on any of the numbers to set the minutes.

    Click on either AM or PM.

    Click OK in the popup.

    You can also click in the text field and type the time directly.

    Time zone

    Click on the down arrow and choose your time zone.

    Su M Tu W Th F Sa

    Click on the day(s) of the week when you want the target to be generated.

    Build even if nothing changed from previous build

    Select this option to always generate output on the specified days and times, even if no source files were changed. If you do not select this option, the target will be generated only if a change is detected in the source files.

    Example

    Let’s say you want to schedule a target named “HTML5 - Top Navigation” to be generated automatically. You want the target to be build every work day (Monday-Friday) at 9 p.m. Pacific time.

 Notes

 What's Next?

After you generate a target, you might want to do any of the following:

  • View You can open the output to see how it looks. See Viewing Output.
  • Open Log You can open the build log to see more details, including any errors or warnings. See Opening the Build Log.
  • Set “Live” or “Keep” If you want to make a particular build accessible to your end users, you can set it as “Live.” You can also set a build to “Keep,” which means that you do not want it to be accidentally deleted; rather, you might want to set it to “Live” later on. See Setting Builds to Live or Keep.
