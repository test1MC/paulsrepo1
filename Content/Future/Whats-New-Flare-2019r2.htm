 Responsive Content on Conditions

When associating conditions with targets in Flare, you have always been able to choose between simply including or excluding them in the output. Starting with this version, you have an additional option. You can choose to include conditions based on media queries. In other words, content tagged with conditions can be set to display only if the screen is a certain size, but will be hidden otherwise. Doing this can make some content easier to read on different devices. See Associating Conditions with Targets and About Mediums and Media Queries.

Features and User Interface Areas Affected

In addition to using this feature in targets, it can also be accessed when working in other areas of the user interface. Following are the features for which you can and cannot set conditions to display by media query:

Limitations

Keep the following in mind if you intend to display conditioned content according to media queries:

  • This feature is not supported in all outputs. It works only for HTML5 targets.
  • Only factory media queries (Web, Tablet, Mobile) are supported and are controlled by the breakpoints specified in the skin or target. Custom media queries do not support this feature.
  • This feature is supported only for conditions applied to content. It does not apply to conditions that are set on folders and files.
 How to Set Conditions by Media Query

Following are steps for setting conditions by media query in the Target Editor. The steps are similar if you are working in the XML Editor, Preview window, or Send Files for Review Wizard; the first few steps would vary, depending on the part of the user interface (for links to each full set of steps, see Features and User Interface Areas Affected).

  2. Double-click the Targets folder. The folder opens, showing the targets in your project.
  3. Double-click the HTML5 target that you want to associate with one or more condition tags. The Target Editor opens to the right.
  4. Click the Conditional Text tab.
  5. Toward the bottom of the tab, make sure the Basic option is selected.
    • Include Select this if you want the content associated with the condition to always display in the target’s output. This is the same as the (default) option.
    • Exclude Select this if you want the content associated with the condition to never display in the target’s output.

    • Display by Media Query Select this if you want the content associated with the condition to display only when the user is viewing the output on a particular screen size (according to the responsive output breakpoints that are set in the target or skin). After selecting this option, choose one or more of the media query check boxes.

      • Web The content will display if the user is viewing it on a “large” screen.
      • Tablet The content will display if the user is viewing it on a “medium” screen, such as an iPad.
      • Mobile The content will display if the user is viewing it on a “small” screen, such as an iPhone.

      If you select all three of the media query check boxes, it has the same effect as choosing the “Include” option.

      The “Display by Media Query” option is not available unless you are working in an HTML5 target. Other target types do not show this option.

    After selecting an option, the Active cell in the grid updates accordingly.

 Additional New Features

These are additional new features added to this release.

508 Compliance xxx

Accessibility improvements have been made for HTML5 output in accordance with 508 compliance standards. These improvements cover keyboard navigation, shortcuts, links, expanding text, and search results.

Keyboard navigation and shortcuts

A link has been added that lets you bypass navigation links and go straight to the main content on the page. As you press the Tab button to scroll through links on the page, the Skip to Main Content link displays in the navigation pane. This link is displayed after you tab past the logo link in the navigation.

This is how the Skip to Main Content link displays in output that uses a top navigation bar.

This is how the Skip to Main Content link displays in output using a tripane navigation.

 Links and expanding text

Submenus can now be accessed through the keyboard. When you click on a main menu option, the submenu can now be displayed when using the Tab button, Enter button, or arrow keys on your keyboard.

 Search results

When results are returned, they are based on the number of searches returned. When you press Enter, the search results display the number of results returned, before displaying the specific results.

The focus on the page is placed on the number of search results returned from the criteria you entered. You can then tab through each of the search results listed below the number of search results listed.

 HTML5 Skin Options

As a result of the new 508 compliance feature, there are new skin components available for your HTML5 output.

The new components are indicated in red rectangles in the screenshot below.

These are the new components in HTML5 skins:

  • Search Filter Selector This component displays the text of the search filter currently selected in the search bar. This field can be edited in the Skin Editor on the UI Text tab.
  • Show Navigation Panel This is the text that will display when you tab to the link that displays the navigation panel. This field can be edited in the Skin Editor on the UI Text tab.
  • Skip to Content This link is used to skip over the navigation and go directly to the main content on a page. This field can be edited in the Skin Editor on the Styles and UI Text tabs.

For more information about editing the values on these fields, see Changing Text Strings in HTML5 Skins.

Collapse Tags in XML Editor

You can now collapse tags while working in the XML editor. This lets you collapse headings, drop-downs, div tags, and snippet blocks.

Span level tags cannot be collapsed at this time in the XML editor. Expanding text blocks cannot be collapsed either.

In the XML editor you see indicators on the left edge to expand or collapse certain blocks. These blocks are expanded by default when you first open a file in the XML editor. There is a - indicator above the respective block in the structure bars area of the editor. You can collapse this block by clicking on the - indicator.

forthcoming imagexxx

You have the option to hide or show the collapse and display indicators in the structure bars. Click on the Show Tags drop-down list in the XML editor to toggle these indicators.

You can have as many items collapsed or expanded on block-level tags within a topic. When block-level tags are collapsed with nested content, the nested content will also be hidden under the collapsed block-level tag.
